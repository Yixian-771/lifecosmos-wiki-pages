# 📘 生命禅院百科馆

# 词条状态追踪表结构规范 v1.0

---

## 一、目的

用于：

* 统一管理所有词条
* 清晰显示完成进度
* 避免遗漏版本
* 避免中英文不同步
* 方便规模扩展

---

## 二、追踪表文件位置

建议在仓库根目录建立：

```
ENTRY_TRACKING.md
```

用于实际记录。

本文件为结构规范说明。

---

## 三、追踪表字段结构

每个词条一行，采用 Markdown 表格结构：

```markdown
| 序号 | 词条中文名 | slug | zh入口 | zh内 | zh友 | zh学 | en入口 | en内 | en友 | en学 | 状态 | 备注 |
```

---

## 四、字段解释

| 字段    | 含义                  |
| ----- | ------------------- |
| 序号    | 自动编号                |
| 词条中文名 | 标准中文名称              |
| slug  | 文件名                 |
| zh入口  | zh/slug.md          |
| zh内   | zh/slug/internal.md |
| zh友   | zh/slug/friendly.md |
| zh学   | zh/slug/academic.md |
| en入口  | en/slug.md          |
| en内   | en/slug/internal.md |
| en友   | en/slug/friendly.md |
| en学   | en/slug/academic.md |
| 状态    | 未开始 / 部分完成 / 完成     |
| 备注    | 特殊说明                |

---

## 五、状态标记规则

统一使用：

```
✔ 已完成
△ 部分完成
✘ 未建立
```

示例：

```markdown
| 01 | 文明3.0 | civilization-3-0 | ✔ | ✔ | ✔ | ✔ | ✔ | ✔ | ✔ | ✔ | 完成 | - |
```

部分完成示例：

```markdown
| 02 | 第二家园模式 | second-home-model | ✔ | ✔ | ✔ | ✘ | ✔ | ✘ | ✘ | ✘ | 部分完成 | 缺英文版本 |
```

---

## 七、工作流程

每创建一个词条：

1. 先在追踪表新增一行
2. 标记全部为 ✘
3. 每完成一个文件，更新 ✔
4. 所有完成后，状态改为“完成”

---

## 八、管理规则

* 任何词条不得跳过追踪表
* 不允许未记录直接发布
* 每周检查一次 incomplete 项

---

## 九、规模扩展预留

未来词条超过 100 个时：

可拆分为：

```
ENTRY_TRACKING_THEORY.md
ENTRY_TRACKING_PRACTICE.md
ENTRY_TRACKING_PEOPLE.md
```

当前 v1.0 统一管理。
